  • SeBootstrap API for programmatic server startup

WildFly Provisioning

  • Traditional: Download full WildFly, deploy WAR
  • Modern: Build exactly what you need
  • Smaller footprint, faster startup
  • Container-friendly deployments
MODERN PROVISIONING (2 minutes): WildFly has completely transformed how you build and deploy applications.
  • The old way: Download a 400MB WildFly zip, unzip it, deploy your WAR to it
  • The new way: Build exactly what your application needs
  • This means smaller footprint, faster startup times
  • And it's perfect for containers - no bloat
KEY POINT: "WildFly isn't a heavyweight app server anymore. You build lean, purpose-built runtimes."

Galleon: Build What You Need

  • Provision custom WildFly servers
  • Choose only the layers you need
  • Significantly reduced disk footprint
  • Maven plugin integration
GALLEON (3-4 minutes with demo): First up - Galleon. This is WildFly's provisioning tool.
  • It lets you provision a custom WildFly server
  • You pick only the layers you need - like ee-core-profile-server, cdi, jpa
  • Dramatically reduces the disk footprint
  • And it's integrated right into the Maven plugin

When to Use What

  • RESTEasy JUnit Extension: Unit tests, REST-only, fast feedback
  • WildFly Testing Tools: WildFly integration tests, simple setup
  • Arquillian: In-container testing or something more generic for other servers
TESTING DECISION GUIDE (2 minutes): "Here's how to choose which testing tool to use."
  • RESTEasy JUnit Extension: Use for unit tests, REST-only, when you want fast feedback "No container, very quick"
  • WildFly Testing Tools: For WildFly integration tests, simple setup "When you need the full server but want it easy"
  • Arquillian: In-container "Arquillian is still great - don't abandon it"
IMPORTANT: "These tools complement Arquillian, they don't replace it." "We still heavily use Arquillian in the WildFly test suite. Both the Jakarta EE and MicroProfile TCK's use Arquillian."
